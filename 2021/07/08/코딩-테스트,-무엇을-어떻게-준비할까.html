코딩테스트의 유형

  • 온라인 코딩 테스트

    • 실제 코딩테스트에서는 횟수 제한이 있을 수 있으니 주의
    • 인터넷에서 참고할 만한 소스코드를 찾더라도 필요한 내용만 확인하여 이를 현재 풀고 있는 문제에 적용이 가능도록 자신만의 소스코드로 표현하는 작성 능력이 중요.
    • 온라인 IDE를 이용할 때는 소스코드가 공개 설정으로 되어 있는지 확인할 필요가 있다.

  • 오프라인 코딩 테스트

    • 대체로 인터넷 검색이 허용되지 않으며 회사에서 제공하는 컴퓨터 환겨에서 바로 시험에 응시
    • 기관이 규정하는 바에 맞게 테스트를 준비하면 됨.

코딩 테스트 준비를 돕는 다양한 서비스

  • 코드업
    • 난이도가 낮은 문제가 많아 처음 공부하는 사람에게 적합
    • 알고리즘을 잘 모르거나 이제 막 시작하려는 독자라면 코드업 사이트의 [문제] - [문제집]에서 [기초 100제]를 꼭 풀어보자.
    • 이 유형을 제대로 학습하면 실전에서 문법이 떠오르지 않아 풀지 못 하는 일은 적을 것.
    • 특히 코딩테스트에서 많은 비중을 차지하는 구현 문제를 해결하는 데 큰 도움이 됨
  • 백준 온라인 저지
    • 유형별 알고리즘을 선택하여 풀수 있음
    • 필자는 책을 읽으며 백준 온라인 저지에서 유형별 알고리즘을 찾아 풀어볼 것을 권장
    • SW 역량 테스트 대비 문제집을 제공하고 있으므로, 대기업 공채를 준비한다면 백준 온라인 저지 문제를 풀어보는 것을 추천
  • 프로그래머스
    • 카카오에 지원하거나 카카오 문제 스타일 확인하고 싶다면 접속해서 반드시 문제를 풀어보자
  • SW Expert Academy
    • 삼성에서 공식적으로 제공하고 있는 알고리즘 학습 사이트
    • 삼성전자의 경우 DFS/BFS를 활용해야 하는 탐색과 시뮬레이션 문제 유형을 자주 출제

어떤 언어가 코딩 테스트에 유리할까?

  • 20년을 기준으로 코테에 가장 유리한 언어는 C++과 파이썬
  • 특히 이제 막 언어를 배우기 시작하는 사람이라면 파이썬이 더 적합
  • C++은 파이썬에 비해 시간적으로 빠르고 효율적이다.
  • 파이썬은 다른 언어에 비해 코드가 짧고 직관적으로 문제를 풀 수 있다.
  • 파이썬은 다른 언어에 비해 큰 숫자, 리스트, 문자열을 처리하는데 매우 편리한 기능을 내장
  • 프로그램 개발에도 파이썬이 더 적합

실습환경

  • 이 책으로 공부하는 동안 컴퓨터에 개발 환경을 설치하지 않고 온라인 실습 환경에서 공부하기를 권장
  • 리플릿, 파이썬 튜터, 온라인 GDB 순으로 온라인 실습 환경을 추천

복잡도

  • 복잡도는 알고리즘의 성능을 나타내는 척도
  • 시간 복잡도 : 알고리즘을 위해 필요한 연산의 횟수
  • 공간 복잡도 : 알고리즘을 위해 필요한 메모리의 양
  • 메모리를 조금 더 많이 사용하는 대신에 반복되는 연산을 생략하거나 더 많은 정보를 관리하면서 계산의 복잡도를 줄일 수 잇다.
  • 즉, 메모리를 더 소모하는 대신에 얻을 수 있는 시간적 이점이 매우 큰 경우가 종종 있음.

시간 복잡도

  • 시간 복잡도를 표현할 때에는 가장 빠르게 증가하는 항만을 고려하는 빅오 표기법을 사용
  • 보통 시간 복잡도에서 연산은 프로그래밍 언어에서 지원하는 사칙 연산, 비교 연산 등과 같은 기본 연산을 의미
  • 아래의 코드에서 연산 횟수는 N에 비례한다.
  • 다른 연산 부분도 있지만, 상대적으로 n이 커짐에 따라서 무시할 수 있을 정도로 작아진다. 따라서, 가장 영향력이 큰 부분은 N에 비례하는 반복문 부분이므로 시간 복잡도를 O(N)이라고 표기한다
  • 만약 소스코드가 내부적으로 다른 함수를 호출한다면 내부 함수의 시간 복잡도까지 고려해야 함
  • 일반적인 코딩 테스트에서는 상수를 고려해야 하는 경우는 적지만, 빅오 표기법이 항상 절대적인 것은 아니라는 점은 기억
array = [3,5,1,2,4]
summary = 0
for x in array : 
    summary += x
print(summary)

    • N의 범위가 100,000인 경우 : 시간 복잡도가 O(Nlog(N))인 알고리즘을 설계하면 문제를 풀 수 있다
    • N의 범위가 10,000,000인 경우 : 시간 복잡도가 O(N)인 알고리즘을 설계하면 문제를 풀 수 있다
